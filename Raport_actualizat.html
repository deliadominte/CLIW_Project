1. Abstarct:

Raportul de fata cuprinde detalii legate de modelarea si stocarea datelor, si de asemenea detalii despre etapele necesare pentru dezvoltarea proiectului "METOO", propus in cadrul laboratorului de Dezvoltarea aplicaţiilor Web la nivel de client.

2. Introducere:

METOO se adreseaza persoanele care studiază/lucrează în localuri comune disponibile într-un areal geografic: biblioteci, cafenele sau spaţii de co-working. Aplicația încurajeaza interacțiunea dintre aceste persoane, pentru a putea studia/lucra împreuna. Istoria a dovedit că oamenii sunt capabili de lucruri mărețe doar dacă colaborează, sau după cum spune Chuck Page, “o frunză ce lucrează de una singură nu face umbră’’.

3. Etapele intermediare ale dezvoltării proiectului:

  • Implementare sistem de Login/Register printr-un cont disponibil pe o retea sociala.
  • Implementarea paginilor de edit/adaugare/vizualizare pentru Profile si Locations.
  • Functionalitate pentru pagina de Discover people.
  • Implementarea functionalitatii de adaugare unei activitati.
  • Functionalitate pentru mecanismul de notificare pentru: mesaje, invitatii de cafea, activitati.
  • Implementarea functionalitatii de compunere/trimitere/vizualizare mesaje.
  • Implementarea functionalitatii de trimitere/vizualizare a invitatiilor la cafea.

Modelarea datelor ce urmeaza a fi cache-uite
Datele ce urmeaza a fi cache-uite vor fi stocate sub forma de perechi de obiecte "Request" si "Response" ( "The caches only store pairs of Request and Response objects, representing HTTP requests and responses, respectively. However, the requests and responses can contain any kind of data that can be transferred over HTTP." : sursa->https://developers.google.com/web/fundamentals/instant-and-offline/web-storage/cache-api)
Exemplu request :

const request = new Request('/images/sample1.jpg');

Exemplu Response:

const imageBlob = new Blob([data], {type: 'image/jpeg'});

const imageResponse = new Response(imageBlob);

const stringResponse = new Response('Hello world');

5. Soluţii de stocare disponibile la nivel de client

Ca solutie de stocare la nivel de client, vom folosi Cache API. Cache API este un sistem pentru stocarea și preluarea cererilor de rețea și a răspunsurilor corespunzătoare. Acesta fost creat pentru a permite service worker-ilor să cacheuiasca cererile de rețea, astfel încât aceștia să poată oferi răspunsuri adecvate chiar și offline. Totuși, acesta poate fi folosit și ca mecanism general de stocare. Mai multe informatii despre acesta se pot gasi de exemplu la adresa: https://developers.google.com/web/fundamentals/instant-and-offline/web-storage/cache-api

6. Testarea si Exploatarea practica:

Dupa ce am implementat functionalitatea proiectului, si acesta este complet din punct de vedere front-end si back-end, am introdus in baza de date utilizatori, locatii si activitati valide dar si invalide, in vederea rularii unui set de teste. Aceste teste le-am realizat pentru a fi siguri ca functionalitatile implementate produc rezultatul dorit. In cazul in care acestea nu au produs rezultatul dorit, sau au aparut erori, am incercat sa rezolvam problemele. Exploatarea practica a solutiei realizate, a fost produsa de catre noi, gandindu-ne la momentele cand am avut nevoie de ajutor, cand am studiat la o materie sau un domeniu in afara facultatii, si ne-ar fi fost mult mai usor sa intrebam pe cineva mai experimentat ca noi. Acest proiect va putea conecta oamenii interesati de aceleasi domenii si incurajeaza invatatul eficient.
